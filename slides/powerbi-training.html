BI ialah gabungan proses, seni bina, dan teknologi yang menukar data mentah kepada maklumat bermakna yang memandu tindakan perniagaan.

Ia menyatukan business analytics, data mining, visualisasi, dan amalan terbaik supaya organisasi membuat keputusan berpacukan data (data-driven).

Contoh NRES: daripada ≈480 rekod aduan mentah → corak musim, beban setiap agensi, dan jumlah kompaun → keputusan penguatkuasaan.

Konsep asas

Apa itu Microsoft Power BI?

Alat BI & visualisasi

Produk visualisasi data interaktif oleh Microsoft untuk menganalisis, membersih, dan menukar data kepada laporan visual.

Sebahagian Power Platform

Bersama Power Apps, Power Automate, Power Pages & Copilot Studio — keluarga alat rendah-kod Microsoft.

Power BI bukan satu aplikasi tunggal — ia koleksi komponen yang bekerja bersama: Power Query, model semantik, visual, Desktop, Service, Gateway, dan apl mudah alih.
Microsoft Power BIKoleksi komponen, bukan satu alat
Ekosistem

Komponen utama Power BI.

Power Query

Enjin transformasi data (ETL). Sambung, bersih, gabung & bentuk data — bahasa M.

Power Pivot

Enjin pemodelan dalam-memori. Bina hubungan & pengiraan DAX → model semantik.

Power View / Visual

Kanvas laporan: carta, peta, kad, matrix & slicer yang interaktif.

Power BI Desktop

Aplikasi Windows percuma untuk bina model + laporan — tempat kerja utama kita.

Power BI Service

Portal awan (SaaS) untuk terbit, kongsi & kolaborasi — kini sebahagian Microsoft Fabric.

Gateway & Mobile

Gateway: segar semula data on-prem. Apl mudah alih: akses di Windows, iOS, Android.

Setiap query ialah senarai Applied Steps yang dirakam mengikut urutan. Klik mana-mana langkah untuk lihat data pada peringkat itu — boleh edit, susun semula, atau buang.

Applied StepsSumber: Bab 4 · Touring Power Query Editor
Power Query · gabung data

Merge vs Append.

Merge → tambah LAJUR

Cantum dua query ikut lajur sepadan (seperti SQL JOIN). 6 Join Kind: Left/Right/Full Outer, Inner, Left/Right Anti. Selepas merge, klik Expand untuk pilih lajur.

Append → tambah BARIS

Tindan baris dua/lebih query jadi satu jadual. Sesuai untuk satukan fail berformat sama (Jan + Feb + Mac) menjadi satu fakta.

NRES: Append eksport aduan bulanan setiap agensi → satu jadual aduan. (Untuk hubungkan fakta↔dimensi kita guna relationship, bukan merge.)
Merge / AppendSumber: Bab 4 · Merging, copying & appending queries
Power Query · bentuk data

Unpivot & disable load.

Unpivot Columns

Data Excel selalu “wide” — satu lajur per bulan (Jan…Dec). Pilih lajur bulan → Transform > Unpivot Columns → jadi 2 lajur: Attribute + Value. Rename Attribute → “Bulan”.

Enable load OFF

Query perantara (hasil Merge/Append) boleh dimatikan Enable load supaya ia tidak jadi jadual berasingan dalam model semantik.

Prinsip model bersih: elak banyak lajur untuk data element sama, elak jadual perantara tak perlu. NRES: kiraan per-bulan → Unpivot → boleh dikait ke Kalendar.

Model · pendalaman

Cardinality & arah penapis.

Cardinality

4 jenis: One-to-one, One-to-many, Many-to-one, Many-to-many. Garis tunjuk 1 di sisi unik, * di sisi banyak. Star schema = One-to-many.

Cross-filter direction

Single: tapisan mengalir satu arah (dim → fakta). Both: dua arah (guna berhati-hati). Hanya satu relationship active (garis pejal) antara dua jadual.

Cipta: seret lajur dalam Model view, atau Manage relationships (Autodetect). NRES: semua dim → aduan ialah One-to-many, cross-filter Single.

Konsep penting

Sukatan vs Lajur Terkira.

Calculated Column

Dikira baris demi baris, disimpan dalam jadual. Guna untuk mengkategori setiap rekod (cth: Kumpulan Tempoh).

Measure (Sukatan)

Dikira mengikut konteks penapis visual, tidak disimpan. Guna untuk agregat (cth: Jumlah Aduan, % Selesai).

Petua: jika ia satu nombor ringkasan yang berubah ikut carta → gunakan measure.
Asas DAXNew column vs New measure
Konsep DAX

Evaluation context: row vs filter.

Row context

Pengiraan hanya “nampak” baris semasa. Default untuk calculated column — cth tarikh_selesai − tarikh_terima bagi setiap aduan.

Filter context

Pengiraan dinilai atas baris yang ditapis oleh visual, slicer & relationship. Fungsi agregat (SUM, COUNTROWS) sentiasa bekerja di sini.

Measure tiada row context — ia bermula dari filter context visual, jadi mesti guna agregat bila merujuk lajur. Inilah sebab measure & calculated column boleh beri jawapan berbeza.
Evaluation contextSumber: Bab 5 · Understanding the context
Langkah 3

Sukatan asas.

Jumlah Aduan = COUNTROWS(aduan)

Aduan Selesai =
    CALCULATE([Jumlah Aduan], aduan[status] = "Selesai")

% Selesai = DIVIDE([Aduan Selesai], [Jumlah Aduan])
CALCULATE menukar konteks penapis · DIVIDE selamat daripada ralat bahagi-sifar.

Mesra pemula

Quick measures: DAX tanpa menulis DAX.

New quick measure buka pane untuk pilih pengiraan biasa — Running total, % of total, Average per category — dan Power BI jana formula DAX automatik.

Hasilnya measure DAX biasa yang boleh dibuka, dipelajari & diubah kemudian. Buku: “particularly helpful when just beginning to use DAX”.

NRES: cipta Running total aduan mengikut Month, atau % of total aduan mengikut agensi — tanpa menulis DAX sendiri.

Gotcha biasa

Sort by Column: susun teks ikut nombor.

Masalah: lajur teks (nama bulan, status) disusun ikut abjad — April, August, December… bukan urutan sebenar.

Selesai: pilih lajur teks → Column tools > Sort by column → pilih lajur nombor padanan (cth Nama Bulan ikut No Bulan 1–12).

Setiap nilai teks mesti petakan ke satu nombor konsisten. Carta kemas kini paksi automatik. NRES: susun status ikut aliran proses (Baru → Dalam Siasatan → Selesai → Ditutup), bukan A–Z.

Corak analisis

Top-N filtering: fokus pemain utama.

Cara

Dalam Filters pane, tukar Filter type medan kepada Top N. Pilih Top atau Bottom, masukkan N (cth 10). Seret measure ke By value sebagai kriteria. Klik Apply filter.

Guna

Visual papar hanya N teratas/terbawah — bukan senarai panjang. Salin visual & tukar Top→Bottom untuk dua sudut. (Bottom boleh papar >N bila ada seri.)

NRES: “Top 5 negeri ikut Jumlah Aduan” atau “Top 3 kategori dengan Jumlah Kompaun (RM) tertinggi”.
